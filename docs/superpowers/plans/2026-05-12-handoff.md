## Key Architecture Decisions

1. **No external dependencies added** — all new code uses stdlib + existing deps (loguru, fastapi, httpx)
2. **All monitoring hooks wrapped in try/except** — never break the request path
3. **Lazy imports** inside try blocks to avoid circular imports (`from kiro.control_panel import control_panel`)
4. **Thread-safe** — RLock on ring buffer and metrics (loguru calls from different threads)
5. **SSE over polling** — `/dashboard/api/events` sends initial snapshot then incremental diffs; 15s keep-alive
6. **Fast-fail on capacity 429** — reads response body for non-stream requests only (can't consume stream body)
7. **RemoteProtocolError retry** — added before the generic `except Exception` in streaming_core's retry loop
8. **`monitor_request_id`** threaded through all streaming/converter call chains as keyword-only `Optional[str] = None`
